# COD哨兵决策说明文档(适用于26赛季RMUL)
## 一、消息
- **0x0001**：当前阶段剩余时间，UNIX 时间，当前比赛阶段
- **0x0002**：比赛结果
- **0x0003**：我方英雄、步兵、哨兵机器人的血量
- **0x0101**：我方补给区的占领状态，中心增益点的占领状态（未被占领、我方占领、敌方占领、双方占领）
- **0x0104**：判罚等级、次数，机器人id
- **0x0201**：哨兵id、血量、血量上限、热量上限、底盘功率上限
- **0x0203**：哨兵机器人X、Y位置坐标，哨兵测速模块的朝向
- **0x0206**：哨兵扣血原因（被弹丸攻击、受撞击或离线），血量变化类型
- **0x0207**：哨兵弹丸射速，弹丸初速度
- **0x0208**：哨兵弹丸允许发弹量，剩余金币数量
- **0x020B**：我方英雄机器人位置X坐标、Y坐标，我方步兵机器人位置X坐标、Y坐标
- **0x020D**：哨兵机器人成功远程兑换血量的次数；哨兵机器人当前是否可以确认免费复活；哨兵机器人当前是否可以兑换立即复活；哨兵机器人当前若兑换立即复活需要花费的金币数；哨兵当前姿态（进攻、防御、移动）

## 二、哨兵功能
- **识别功能**：识别我方是否占领中心增益区
- **触发条件**：接受0x0101消息，1是我方占领，2是敌方占领，3是双方占领。如果是双方占领，根据消息的转变来判断，如果是从1变为3，则为我方占领，否者敌方占领。

- **占领中心增益区功能**：导航到中心增益区的中心位置，实现占领
- **触发条件**：通过0x0002消息确认比赛一开始 或 敌方占领中心增益区。

- **定位功能**：获取哨兵，英雄，步兵机器人的位置X坐标，Y坐标
- **触发条件**：通过0x0203消息获取哨兵机器人X、Y、位置坐标，哨兵测速模块的朝向。通过0x020B消息获取英雄和步兵机器人X、Y、位置坐标。

- **1号站位功能**：导航到航点1站位，卡敌方位置，给敌方上压力
- **触发条件**：哨兵PH>50%，我方英雄或步兵占领了增益区。

- **2号站位功能**：导航到航点2站位，卡敌方位置，抵制敌方机器人的追击
- **触发条件**：我方英雄和步兵都处于低血量状态。

- **巡逻功能**：依照设置的巡逻路线进行移动（强势压制）
- **触发条件**：我方占领中心增益区。

- **补血功能**：回家补血
- **触发条件**：哨兵PH<30%

- **协助功能**：在中心增益区执行小规模的移动
- **触发条件**：我方占领中心增益区，同时有另一台也在。

- **被动防御**：小陀螺加速旋转
- **触发条件**：通过0x0206消息，接收到是被弹丸攻击所受的伤害。

- **复活功能**：复活哨兵机器人
- **触发条件**：通过0x020D消息确认可以免费复活。

- **停止射击功能**：停止一切要射击的功能
- **触发条件**：通过0x0208确定允发弹量为0，或通过0x0201确认热量大于等于热量上限、底盘功率大于等于底盘功率上限。

---

## 三、决策：
- **主要核心策略：站住中心增益区**
- 比赛一开始去往中心增益区占领，然后根据状态执行各个功能，补血功能优先级最高，其次是占领中心增益区。大部分时间哨兵处于中心增益区的中心。
